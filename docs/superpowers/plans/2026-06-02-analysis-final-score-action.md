# Analysis final_score Action Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an analysis-engine special action `final_score` that returns GTP-style final-score estimates without changing normal analysis responses.

**Architecture:** Extend the existing analysis request queue with a request kind so `final_score` uses the same parsing, priority, worker pool, and termination machinery as normal analysis. Add a local final-score helper that mirrors GTP `computeAnticipatedWinnerAndScore`, then route worker execution to either normal analysis JSON or final-score JSON.

**Tech Stack:** C++17, KataGo analysis engine, `AsyncBot`, `PlayUtils::computeLead`, nlohmann JSON, shell-driven command tests.

- [ ] **Step 1: Add a response formatter helper**

Near the other local lambdas inside `MainCmds::analysis`, before `analysisLoop`, add a helper lambda. Include a comment because repository instructions require function comments:

```cpp
  // Purpose: Format a white-minus-black final score into the same string form as GTP final_score.
  // Params: winner is the winning player or C_EMPTY for jigo; finalWhiteMinusBlackScore is positive when white is ahead.
  // Return: "B+N.N", "W+N.N", or "0".
  auto formatFinalScore = [](Player winner, double finalWhiteMinusBlackScore) {
    if(winner == C_EMPTY)
      return string("0");
    if(winner == P_BLACK)
      return string("B+") + Global::strprintf("%.1f",-finalWhiteMinusBlackScore);
    if(winner == P_WHITE)
      return string("W+") + Global::strprintf("%.1f",finalWhiteMinusBlackScore);
    ASSERT_UNREACHABLE;
    return string("0");
  };
```

- [ ] **Step 2: Add final-score computation helper**

Add a second lambda near `formatFinalScore`:

```cpp
  // Purpose: Compute the GTP-style final score for one analysis-engine request position.
  // Params: bot owns the search resources; request supplies board, history, player, and params; response receives JSON fields.
  // Return: None. Throws only via existing search utilities if the underlying search cannot produce values.
  auto computeFinalScoreResponse = [&formatFinalScore](AsyncBot* bot, const AnalyzeRequest* request, json& response) {
    SearchParams finalScoreParams = request->params;
    finalScoreParams.playoutDoublingAdvantage = 0.0;
    finalScoreParams.conservativePass = true;
    finalScoreParams.humanSLChosenMoveProp = 0.0;
    finalScoreParams.humanSLRootExploreProbWeightful = 0.0;
    finalScoreParams.humanSLRootExploreProbWeightless = 0.0;
    finalScoreParams.humanSLPlaExploreProbWeightful = 0.0;
    finalScoreParams.humanSLPlaExploreProbWeightless = 0.0;
    finalScoreParams.humanSLOppExploreProbWeightful = 0.0;
    finalScoreParams.humanSLOppExploreProbWeightless = 0.0;
    finalScoreParams.antiMirror = false;
    finalScoreParams.avoidRepeatedPatternUtility = 0.0;

    Player winner = C_EMPTY;
    double finalWhiteMinusBlackScore = 0.0;
    string scoreSource;

    Board board = request->board;
    BoardHistory hist = request->hist;
    Player pla = request->nextPla;

    bot->setParams(finalScoreParams);
    bot->setPosition(pla,board,hist);
    vector<int> noAvoidMovesBlack;
    vector<int> noAvoidMovesWhite;
    vector<Loc> noIncludeMovesBlack;
    vector<Loc> noIncludeMovesWhite;
    bot->setAlwaysIncludeOwnerMap(false);
    bot->setAvoidMoveUntilByLoc(noAvoidMovesBlack,noAvoidMovesWhite);
    bot->setIncludeMoves(noIncludeMovesBlack,noIncludeMovesWhite);

    if(hist.isGameFinished && (
         (hist.rules.scoringRule == Rules::SCORING_AREA && !hist.rules.friendlyPassOk) ||
         (hist.rules.scoringRule == Rules::SCORING_TERRITORY)
       )
    ) {
      winner = hist.winner;
      finalWhiteMinusBlackScore = hist.finalWhiteMinusBlackScore;
      scoreSource = "finished";
    }
    else {
      int64_t numVisits = std::max((int64_t)50, (int64_t)finalScoreParams.numThreads * 10);
      double lead = PlayUtils::computeLead(bot->getSearchStopAndWait(),NULL,board,hist,pla,numVisits,OtherGameProperties());
      if(hist.rules.gameResultWillBeInteger())
        lead = round(lead);
      else
        lead = round(lead+0.5)-0.5;
      finalWhiteMinusBlackScore = lead;
      winner = lead > 0 ? P_WHITE : lead < 0 ? P_BLACK : C_EMPTY;
      scoreSource = "estimated";
    }

    response["winner"] = winner == C_EMPTY ? "0" : PlayerIO::playerToStringShort(winner);
    response["finalScore"] = formatFinalScore(winner, finalWhiteMinusBlackScore);
    response["finalWhiteMinusBlackScore"] = Global::roundDynamic(finalWhiteMinusBlackScore, 8);
    response["scoreSource"] = scoreSource;
  };
```

## Task 4: Document final_score Special Action

**Files:**
- Modify: `docs/Analysis_Engine.md`

- [ ] **Step 1: Add documentation section**

In `docs/Analysis_Engine.md`, add a new special action section after `clear_cache` and before `terminate`:

````markdown
#### final_score

Requests that KataGo compute a GTP-style final score for one or more requested turns without returning normal analysis data. This action is queued with normal analysis work because it may run several short searches to estimate the score of unfinished positions.

Required and optional position fields are the same as a normal analysis request: `boardXSize`, `boardYSize`, `initialStones`, `moves`, `initialPlayer`, `rules`, `komi`, `whiteHandicapBonus`, `analyzeTurns`, `priorities`, and `overrideSettings`.

Required fields:

   * `id (string)`: Required. An arbitrary string identifier for this query.
   * `action (string)`: Required. Should be the string `final_score`.

Example:
```json
{"id":"foo","action":"final_score","initialStones":[],"moves":[["B","D4"],["W","Q16"]],"rules":"chinese","komi":7.5,"boardXSize":19,"boardYSize":19}
```

The response contains one JSON object for each requested turn:

   * `id`: The same id string that was provided on the query.
   * `action`: The string `final_score`.
   * `turnNumber`: The turn number being scored.
   * `isDuringSearch`: Always false.
   * `winner`: `"B"`, `"W"`, or `"0"`.
   * `finalScore`: A GTP-style score string such as `"B+2.5"`, `"W+3.5"`, or `"0"`.
   * `finalWhiteMinusBlackScore`: The estimated final score as white minus black, positive when white is ahead.
   * `scoreSource`: `"finished"` if the game history already has a final score, otherwise `"estimated"`.

Example response:
```json
{"action":"final_score","finalScore":"W+3.5","finalWhiteMinusBlackScore":3.5,"id":"foo","isDuringSearch":false,"scoreSource":"estimated","turnNumber":2,"winner":"W"}
```
````
